• $ git config --global user.name "[ім'я]" - встановлює ім'я, яке буде відображатися у полі автора у виконуваних коммітів

  • $ git config --global user.email "[адреса електронної пошти]" - встановлює адресу електронної пошти, який буде відображатися у інформації про виконувані комміти

  • $ git clone [url-адреса] - завантажує репозиторій разом зі всією його історією змін

  • $ git status - перелічує всі нові або змінені файли, які потребують фіксації

  • $ git add [файл] - індексує вказаний файл для наступного комміту

  • $ git reset [файл] - відміняє індексацію вказаного файлу, при цьому зберігає його зміст

  • $ git commit -m "[повідомлення з описом]" - фіксує проіндексовані зміни і зберігає їх в історію версій

  • $ git branch [ім'я гілки] - створює нову гілку

  • $ git switch -c [ім'я гілки] - переключається на обрану гілку і оновлює робочу директорію до її стану

  • $ git merge [ім'я гілки] - вносить зміни вказаної гілки у поточну гілку

  • $ git branch -d [ім'я гілки] - видаляє обрану гілку

  • $ git log - історія коммітів для поточної гілки

  • $ git reset [комміт] - відміняє всі комміти після заданого, залишаючи всі зміни у робочій директорії

  • $ git reset --hard [комміт] - скидає всю історію разом із станом робочої директорії до вказаного комміта

  • $ git fetch [віддалений репозиторій] - завантажує всю історію з віддаленого репозиторія

  • $ git merge [віддалений репозиторій]/[гілка] - вносить зміни з гілки віддаленого репозиторія в поточну гілку локального репозиторія

  • $ git push [віддалений репозиторій] [гілка] - завантажує всі зміни локальної гілки у віддалений репозиторій

  • $ git pull - завантажує історію з віддаленого репозиторія і об'єднує її з локальною. pull = fetch + merge
